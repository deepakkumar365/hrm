================================================================================
                        🔐 SECURITY NOTES
================================================================================

✅ WHAT YOU SHOULD KNOW:

1. Database Credentials:
   • Stored in .env file (not in version control)
   • Keep .env file safe and confidential
   • Different URLs for dev and production

2. Passwords:
   • Hashed in database using Scrypt (one-way encryption)
   • Cannot be retrieved (only reset)
   • After migration, reset to temporary value
   • Users forced to change on first login

3. Backups:
   • Store in safe location (not public cloud)
   • Backup files contain sensitive data
   • Protect access to backup files
   • Test restore procedure

4. Post-Migration:
   • No users have permanent password until they change it
   • Session timeouts configured (2 hours)
   • HTTPS enabled in production
   • Consider 2FA for admin accounts
